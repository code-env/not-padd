Sometimes, two components need to share the same data. The best way to do that in React is by **lifting the state up** to their common parent.

Here’s an example:

```jsx
import { useState } from "react";

function TemperatureInput({ value, onChange }) {
  return (
    <input
      type="number"
      value={value}
      onChange={(e) => onChange(e.target.value)}
      placeholder="Enter temperature"
    />
  );
}

export default function App() {
  const [temp, setTemp] = useState("");

  return (
    <div style={{ textAlign: "center", marginTop: "40px" }}>
      <TemperatureInput value={temp} onChange={setTemp} />
      <p>{temp && `Temperature: ${temp}°C`}</p>
    </div>
  );
}
```

### How It Works

- The state `temp` is stored in the parent component (`App`).

- The child component (`TemperatureInput`) receives the value and the updater.

- Any change in the input updates the shared state.

Lifting state up keeps data consistent across related components nothing.
